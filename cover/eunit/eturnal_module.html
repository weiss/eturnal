19 %%% @doc An eturnal module adds functionality to the eturnal server. It is to be
20 %%% named `mod_foo', where `foo' describes the added functionality. The module
21 %%% may export `start/0', `stop/0', `handle_event/2', and `options/0' functions.
22 %%%
23 %%% If a `start/0' callback is exported, it must return `ok' or `{ok, Events}',
24 %%% where `Events' is either a single {@link event()} or a list of {@link
25 %%% events()} the module is interested in. Currently, the following events may
26 %%% be triggered: `stun_query', `turn_session_start', and `turn_session_stop'.
27 %%%
28 %%% If a `start/0' function is exported and subscribes to one or more events, a
29 %%% `handle_event/2' callback <em>must</em> be exported as well. It is called
30 %%% with the {@link event()} name as the first argument and an {@link info()}
31 %%% map with related data as the second. The contents of that map depend on the
32 %%% event. Note that the `handle_event/2' function is executed in the context of
33 %%% the process handling the STUN/TURN session, so it should never block. If it
34 %%% might, and/or if it needs some state, one or more handler processes must be
35 %%% created.
36 %%%
37 %%% The `options/0' callback returns an {@link options()} tuple with two
38 %%% elements. The first is a map of module configuration options, where the keys
39 %%% are the {@link option()} names and the values are functions that validate
40 %%% the option values. Those functions are returned by the <a
41 %%% href="https://hex.pm/packages/yval">yval</a> library, see the documentation
42 %%% for the list of <a href="https://hexdocs.pm/yval/yval.html#index">available
43 %%% validators</a>. The second element is a list of optional tuples to specify
44 %%% any `{required, [Options]}' and/or `{defaults, #{Option => Value}}'. For
45 %%% example:
46 %%%
47 %%% ```
48 %%% options() ->
49 %%% {#{threshold => yval:pos_int()},
50 %%% [{defaults,
51 %%% #{threshold => 42}}]}.
52 %%% '''
53 %%%
54 %%% The option values are queried by calling {@link eturnal_module:get_opt/2}
55 %%% with the module name as the first and the {@link option()} name as the
56 %%% second argument. Note that the lookup is very efficient, so there's no point
57 %%% in saving option values into some state. If the module has no configuration
58 %%% options, the `options/0' function may be omitted.
59 %%%
60 %%% The optional `stop/0' callback must return `ok'. Note that the `start/0' and
61 %%% `stop/0' functions might not just be called on eturnal startup and shutdown,
62 %%% but also on configuration reloads.
63 %%%
64 %%% If the module depends on other applications, those must be added to the
65 %%% `rebar.config' file, but not to the app file. They are to be started by
66 %%% calling {@link eturnal_module:ensure_deps/2}, where the first argument is
67 %%% the module name and the second is a list of dependency names. Note that
68 %%% there's no need to list transitive dependencies.
69 %%%
70 %%% The module is enabled by adding its configuration to the `modules' section
